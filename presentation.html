Introduction

ICC Companion is a web-based student portal built for ICON Commerce College. It digitizes daily academic operations — from attendance tracking to resource sharing — into a single unified platform for students, teachers, and administrators.

🎯

Objective

Replace manual, paper-based processes with a real-time digital system that improves transparency and efficiency.

👥

Target Users

Students (BCA, BBA, BA, B.Com), Faculty members who manage classes, and College administrators.

Problem Statement

Existing System Issues

  • Attendance tracked via paper registers — prone to errors and delays.
  • Important notices shared through WhatsApp groups — easily missed.
  • No centralized place for study materials, syllabus, or question papers.
  • Students unaware of their attendance percentage until semester end.

Problem Definition

"To design and develop a secure, user-friendly web portal that automates attendance tracking, centralizes academic resources, and provides real-time analytics for students and faculty at ICON Commerce College."

Proposed System & Objectives

ICC Companion provides the following core features across Student, Admin, and Principal portals:

📊

Attendance Tracking

Subject-wise real-time attendance with color-coded bars and 75% threshold alerts.

📚

Resource Library

Upload & download syllabus, notes, books, and materials — filtered by department & semester.

📅

Exam Schedules

View sessional and final exam timetables with subject, date, time, and room details.

📋

Class Routines

Department-wise weekly class routine PDFs uploaded and managed by administrators.

🔍

Lost & Found

Report lost items or post found items with category, location, and claim status tracking.

📢

Announcements

Priority-based notices (High/Medium/Low) targeted by department and semester.

System Architecture

The application follows a 3-Tier Architecture (Client → Server → Database).

🖥️ PresentationHTML · CSS · JS
⟷
⚙️ ApplicationPHP · XAMPP Apache
⟷
🗄️ DataMySQL · phpMyAdmin
  • Client: Responsive web pages served to any browser (mobile or desktop).
  • Server: PHP handles authentication, CRUD operations, and file uploads.
  • Database: MySQL stores students, attendance, resources, announcements, and more.

Database Design

The campus_portal database contains 11 normalized tables with foreign key constraints + 1 view.

departments

dept_id · dept_code · dept_name

students

student_id · roll_number · name · dob · dept_code · semester

admins

admin_id · username · password · name · role

subjects

subject_id · subject_code · subject_name · dept_code · semester

attendance

attendance_id · roll_number (FK) · subject_id (FK) · date · status

resources

resource_id · title · type · file_url · dept_code · semester

announcements

announcement_id · title · priority · target_dept · target_semester

exams

exam_id · subject_id (FK) · exam_date · time · room · exam_type

exam_schedules

schedule_id · schedule_type (sessional/final) · file_url

class_routines

routine_id · title · file_url · dept_code · semester

lost_found

item_id · title · category · item_type · location · status

student_attendance_summary

VIEW — pre-computed attendance percentages per student per subject

Security Features

  • Role-Based Access: Students, Faculty, HOD, and Principal each have separate permissions and restricted views.
  • Session Management: PHP sessions to maintain login state and prevent unauthorized access.
  • Input Validation: Server-side validation on all forms to prevent SQL injection and XSS.
  • Secure File Uploads: File type and size restrictions on PDF and image uploads.
  • Password Protection: Admin credentials stored securely in the database.
  • Student Auth: Login via Roll Number + Date of Birth — no complex passwords needed.
  • Unique Constraints: Database-level enforcement to prevent duplicate attendance entries.

Limitations & Future Scope

Current Limitations

  • Currently runs on a local XAMPP server setup.
  • No email or SMS notification system.

Future Enhancements

Conclusion

ICC Companion successfully digitizes the academic workflow at ICON Commerce College — replacing manual attendance registers, scattered WhatsApp notices, and disorganized resources with a single, transparent, real-time platform for students, faculty, and administrators.
